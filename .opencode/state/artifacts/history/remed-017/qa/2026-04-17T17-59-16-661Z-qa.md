---

## Verdict: FAIL — smoke_test returns tooling_parse_warning instead of syntax_error

The fix described in the implementation and review artifacts (reordering `isGodotFatalDiagnosticOutput()` before `isClassNameReloadParseWarning()`) has NOT been correctly applied to `.opencode/tools/smoke_test.ts`. When `godot4 --headless --path . --quit` is run, the fatal script-load diagnostic `ERROR: Failed to load script "res://scripts/wave_spawner.gd" with error "Parse error".` is NOT being classified as `syntax_error` — instead it is classified as `tooling_parse_warning`, and the overall smoke test PASSES.

This is the EXACT defect that REMED-017 was supposed to fix (EXEC-GODOT-006: smoke_test reports PASS despite fatal script-load diagnostic in stderr).

### 4. Code ordering analysis

**File:** `.opencode/tools/smoke_test.ts` (lines 605-608)

**Current code observed:**
```typescript
if (args.exitCode === 0) {
  if (isGodotFatalDiagnosticOutput(output) || isSyntaxErrorOutput(output)) return "syntax_error"
  if (isClassNameReloadParseWarning(output, args.exitCode)) return "tooling_parse_warning"
  return undefined
}
```

**Analysis:**
- `isGodotFatalDiagnosticOutput()` should fire first and match `Failed to load script "res:\/\//` pattern
- But the actual classification is `tooling_parse_warning`, meaning either:
  1. `isGodotFatalDiagnosticOutput()` is not matching the pattern, OR
  2. The code shown here doesn't reflect what's actually running in the smoke_test tool

The ordering (fatal first, then class_name) is correct in the source file, but the behavior is incorrect — `isClassNameReloadParseWarning()` is returning `tooling_parse_warning` instead of `isGodotFatalDiagnosticOutput()` returning `syntax_error`.
