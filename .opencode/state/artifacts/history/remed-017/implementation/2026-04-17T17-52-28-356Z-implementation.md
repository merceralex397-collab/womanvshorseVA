# Implementation — REMED-017

## Ticket
- ID: REMED-017
- Title: Godot release smoke artifact reports PASS despite recorded runtime or command-failure evidence
- Stage: implementation
- Status: in_progress

## What Was Changed

Reordered the classification checks inside `classifyCommandFailure()` in `.opencode/tools/smoke_test.ts` so that `isGodotFatalDiagnosticOutput()` is evaluated **before** `isClassNameReloadParseWarning()` within the `exit_code === 0` branch.

### Before (lines 605–608)
```typescript
  if (args.exitCode === 0) {
    if (isClassNameReloadParseWarning(output, args.exitCode)) return "tooling_parse_warning"
    if (isGodotFatalDiagnosticOutput(output) || isSyntaxErrorOutput(output)) return "syntax_error"
    return undefined
  }
```

### After (lines 605–608)
```typescript
  if (args.exitCode === 0) {
    if (isGodotFatalDiagnosticOutput(output) || isSyntaxErrorOutput(output)) return "syntax_error"
    if (isClassNameReloadParseWarning(output, args.exitCode)) return "tooling_parse_warning"
    return undefined
  }
```

## Why This Fix Works

- `isGodotFatalDiagnosticOutput()` catches `Failed to load script` / `Parse Error` patterns that indicate genuine script-load failures
- `isClassNameReloadParseWarning()` catches the specific class_name RELOAD noise that Godot emits even when exit code is 0
- Previously, `isClassNameReloadParseWarning()` fired first and returned `tooling_parse_warning`, silently consuming fatal diagnostic output that should have blocked the smoke test
- The new ordering checks fatal diagnostics first; if a command's stderr contains `Failed to load script`, it is classified as `syntax_error` and `commandBlocksPass()` returns `true`, causing the smoke test to fail as expected
- Only commands whose stderr contains class_name RELOAD noise **without** fatal script-load patterns fall through to `isClassNameReloadParseWarning()` → `tooling_parse_warning` (non-blocking)
