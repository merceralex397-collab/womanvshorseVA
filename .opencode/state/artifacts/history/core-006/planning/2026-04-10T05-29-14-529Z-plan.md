# Planning Artifact — CORE-006: Create enemy variants (brown, black, war horse, boss)

## 1. Overview and Goal

Create four distinct enemy variant scripts extending `EnemyBase`, each with unique visual and stat profiles matching the canonical spec. The variants are: Brown (slow, 1HP), Black (fast, 1HP with speed-line visual), War Horse (thicker, slow, 3HP), and Boss (large, 10HP with pulsing gold modulate). The wave spawner will be updated to instantiate the correct variant by type name.

## 2. Files to Create

| File | Purpose |
|------|---------|
| `scripts/enemy_brown.gd` | Brown enemy variant (Color(0.6,0.4,0.2), 25x35, speed=80, HP=1) |
| `scripts/enemy_black.gd` | Black enemy variant (Color(0.2,0.2,0.2), 25x35, speed=150, HP=1) + speed lines |
| `scripts/enemy_war.gd` | War horse variant (Color(0.8,0.2,0.2), 35x50, speed=60, HP=3) |
| `scripts/enemy_boss.gd` | Boss variant (Color(1.0,0.84,0.0), 50x65, speed=100, HP=10) + pulsing modulate |

## 3. Files to Modify

| File | Change |
|------|--------|
| `scripts/wave_spawner.gd` | Replace `EnemyBase.new()` + manual property assignment with per-type variant instantiation via factory-like `get_enemy_scene(type)` match |

## 4. Step-by-Step Implementation Plan

### Step 1: Create `enemy_brown.gd`
- Extend `EnemyBase`
- Set `@export` defaults: `max_health=1`, `speed=80.0`, `body_color=Color(0.6,0.4,0.2)`, `body_size=Vector2(25,35)`
- Set `type = "brown"`
- No special visuals (base class Polygon2D body + head is sufficient)

### Step 2: Create `enemy_black.gd`
- Extend `EnemyBase`
- Set `@export` defaults: `max_health=1`, `speed=150.0`, `body_color=Color(0.2,0.2,0.2)`, `body_size=Vector2(25,35)`
- Set `type = "black"`
- Override `_draw()` to call `super` then draw 3–4 short speed-line strokes behind the body (using `+body_size.x * 0.6` offset in the opposite direction of facing)
- Lines are `Color(0.4,0.4,0.4)` (subtle grey) at ~60% body width length

### Step 3: Create `enemy_war.gd`
- Extend `EnemyBase`
- Set `@export` defaults: `max_health=3`, `speed=60.0`, `body_color=Color(0.8,0.2,0.2)`, `body_size=Vector2(35,50)`
- Set `type = "war"`
- Thicker body achieved via larger `body_size` — base class handles Polygon2D rendering automatically

### Step 4: Create `enemy_boss.gd`
- Extend `EnemyBase`
- Set `@export` defaults: `max_health=10`, `speed=100.0`, `body_color=Color(1.0,0.84,0.0)`, `body_size=Vector2(50,65)`
- Set `type = "boss"`
- Override `_process(delta)` to oscillate `modulate` between `Color(1.0,0.84,0.0)` and `Color(1.3,1.1,0.3)` using `sin(Time.get_ticks_msec() * 0.003) * 0.3 + 0.85` as a brightness scalar on each channel
- Pulsing period ~2 seconds (smooth sine curve)

### Step 5: Update `wave_spawner.gd` — `_create_enemy()`
- Replace current match statement that creates `EnemyBase.new()` and manually sets properties
- Instead use a dictionary mapping type strings to PackedScene or script paths, or a `match` that returns the correct script class
- Since Godot GDScript can't directly `new()` a script class by string without `load()`, use `match type: "brown": return enemy_brown.new()` etc.
- Add "war" case to the match (currently missing from wave spawner)

### Step 6: Verify `wave_spawner.gd` — `_get_wave_composition()`
- Confirm "war" is not yet used in wave composition (verified — war horse not in spec yet, but `type="war"` should still be set correctly on the variant)
- No changes needed to composition logic; war is future-wave content

## 7. Risks and Mitigations

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| Speed lines `_draw()` uses wrong local coordinate space (flipped or off-center) | Low | Visual bug | Lines drawn relative to body center at `body_size.x * 0.3` behind facing direction using `rotation` |
| Boss sine-wave modulate affects children unexpectedly | Low | Visual flash artifacts | Only set `modulate` on `self.modulate`, not on child nodes; base class sets `modulate = Color.WHITE` during damage flash which overrides briefly — acceptable |
| Wave spawner can't load script by string at runtime | Low | Crash on wave start | Use explicit `match type: "brown": return enemy_brown.new()` pattern instead of dynamic `load()` |
| War horse body looks identical to brown due to same Polygon2D shape | Low | No gameplay impact | Visual differentiation via color and size — accepted |
| Enemy variants reuse same scene file causing collision layer mismatch | N/A | — | Each variant script is a fresh instance; `collision_layer=2` set in `EnemyBase._ready()` — inherited correctly |
