## Findings

### Signal Connection Issues (CRITICAL)

**main.gd lines 63-78 have broken signal connection checks:**

```gdscript
if player and "health_changed" in player:
    if not player.connect("health_changed", Callable(hud_script, "update_health")):
        pass
```

Problems:
1. `"health_changed" in player` checks if the **string** `"health_changed"` is a key in the player object — this is not how you check for signals in Godot 4
2. `connect()` returns an `Error` (int), so `not Error` is always truthy when Error is 0 (OK) — the `pass` suppresses errors but silently succeeds even when wrong
3. The proper Godot 4 way to check signal existence is to verify the signal exists via `has_signal()`

**Impact:** If the player node is missing the `health_changed` signal or the connection syntax is wrong, no error will be raised and the HUD won't update.

**Correct pattern:**
```gdscript
if player.has_signal("health_changed"):
    player.connect("health_changed", Callable(hud_script, "update_health"))
```

The same pattern applies to `wave_started` and `score_changed` checks.

## Review Decision

**REJECT** — Signal connection logic is broken and will prevent the HUD from updating correctly at runtime.

**Root cause:** String-membership check on player object does not validate signal existence. Boolean inversion of Error return value is wrong.

**Required fix:** Change signal connection pattern to use `has_signal()` and direct `connect()` call without the broken boolean inversion.
