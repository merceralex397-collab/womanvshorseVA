# Implementation for REMED-015

## Ticket

- **ID:** REMED-015
- **Title:** smoke_test tool classifies Godot class_name reload parse errors as syntax_error despite exit 0
- **Stage:** implementation
- **Wave:** 22

## Summary

Implemented the fix in `.opencode/tools/smoke_test.ts` to classify Godot class_name reload parse errors as `tooling_parse_warning` instead of `syntax_error` when exit code is 0. This allows FINISH-VALIDATE-001's smoke test to pass despite the tooling artifact stderr.

## Changes Made to `.opencode/tools/smoke_test.ts`

### Change 1: Add `tooling_parse_warning` to `failure_classification` type union

**Location:** Line 44

**Before:**
```typescript
failure_classification?: "missing_executable" | "permission_restriction" | "syntax_error" | "test_failure" | "configuration_error" | "command_error"
```

**After:**
```typescript
failure_classification?: "missing_executable" | "permission_restriction" | "syntax_error" | "test_failure" | "configuration_error" | "command_error" | "tooling_parse_warning"
```

### Change 2: Add `isClassNameReloadParseWarning` helper function

**Location:** Lines 586-591 (after `isGodotFatalDiagnosticOutput`)

```typescript
function isClassNameReloadParseWarning(stderr: string, exitCode: number): boolean {
  if (exitCode !== 0) return false
  if (!stderr.includes("Could not parse global class")) return false
  if (!stderr.includes("Could not resolve class") && !stderr.includes("GDScript::reload")) return false
  return true
}
```

**Logic:** Returns true when ALL three conditions are met:
1. `exitCode === 0`
2. stderr contains `"Could not parse global class"`
3. stderr contains `"Could not resolve class"` OR `"GDScript::reload"`

### Change 3: Update `classifyCommandFailure` to check `isClassNameReloadParseWarning`

**Location:** Lines 604-607

**Before:**
```typescript
  if (args.exitCode === 0) {
    if (isGodotFatalDiagnosticOutput(output)) return "syntax_error"
    return undefined
  }
```

**After:**
```typescript
  if (args.exitCode === 0) {
    if (isClassNameReloadParseWarning(args.stderr, args.exitCode)) return "tooling_parse_warning"
    if (isGodotFatalDiagnosticOutput(output)) return "syntax_error"
    return undefined
  }
```

**Logic:** Before returning `"syntax_error"` for exit_code===0, now checks if the error matches the class_name reload pattern and returns `"tooling_parse_warning"` instead.

### Change 4: Verify `commandBlocksPass` does NOT block on `tooling_parse_warning`

**Location:** Lines 623-629

```typescript
function commandBlocksPass(command: Pick<CommandResult, "exit_code" | "failure_classification">): boolean {
  return (
    command.exit_code !== 0
    || command.failure_classification === "syntax_error"
    || command.failure_classification === "configuration_error"
  )
}
```

**Verification:** `tooling_parse_warning` is NOT in the blocking list. Only `syntax_error` and `configuration_error` block the pass condition. This confirms `tooling_parse_warning` is non-blocking per the REMED-014 plan-review note.
