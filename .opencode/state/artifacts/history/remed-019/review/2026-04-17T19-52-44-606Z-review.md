## Technical Analysis

### Problem

The `isGodotFatalDiagnosticOutput()` function in `.opencode/tools/smoke_test.ts` contained a regex bug at line 584 that prevented detection of Godot fatal script-load errors:

```typescript
// BUGGY (old):
|| /Failed to load script "res:\/\//i.test(output)
```

The pattern required literal `\/\/` (escaped forward slashes) but Godot outputs `res://` (unescaped `://`), so the pattern never matched.

### Fix Applied

```typescript
// FIXED (new):
|| /Failed to load script/i.test(output)
```

The simplified broad pattern catches all "Failed to load script" variants regardless of path format.

### Verification


### Classification Effect

With the fixed regex `/Failed to load script/i`:
1. `isGodotFatalDiagnosticOutput()` correctly matches the fatal script-load diagnostic in stderr
2. `failure_classification` = `syntax_error` (blocking)
3. `commandBlocksPass()` returns `true`
4. Overall smoke test result: **FAIL** with `failure_classification: syntax_error`

This means EXEC-GODOT-006 is now **correctly detected** — the smoke test no longer falsely reports PASS when Godot emits fatal script-load diagnostics alongside exit code 0.
