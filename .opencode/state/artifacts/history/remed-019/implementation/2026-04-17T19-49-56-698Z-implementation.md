# Implementation — REMED-019

## Fix Applied

**File**: `.opencode/tools/smoke_test.ts` line 584

**Change**: Simplified the `isGodotFatalDiagnosticOutput()` regex from:
```typescript
|| /Failed to load script "res:\/\//i.test(output)
```
to:
```typescript
|| /Failed to load script/i.test(output)
```

**Why**: The original regex required literal `\/\/` (escaped forward slashes) but Godot outputs `res://` (unescaped colons and slashes). The simplified broad pattern catches all "Failed to load script" variants regardless of path format.

## Verification

**Command**: `godot4 --headless --path . --quit`

**Exit code**: 0

**Relevant stderr excerpt**:
```
ERROR: Failed to load script "res://scripts/wave_spawner.gd" with error "Parse error".
   at: load (modules/gdscript/gdscript.cpp:2907)
```

The new regex `/Failed to load script/i` matches this output, so Godot fatal script-load errors with `exit_code===0` will now be correctly classified as `syntax_error` (blocking) instead of slipping through as `tooling_parse_warning` (non-blocking).

## Classification Effect

With this fix in `classifyCommandFailure()`:
1. `isGodotFatalDiagnosticOutput()` fires before `isClassNameReloadParseWarning()`
2. The `Failed to load script` pattern is matched by the broadened regex
3. `failure_classification` = `syntax_error` (blocking)
4. Smoke test **FAILS** when Godot reports script-load failures, preventing false PASS artifacts

This resolves EXEC-GODOT-006.
